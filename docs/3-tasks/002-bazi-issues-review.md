## 背景

Amber Compass 已进入 Phase 2：八字模块验证与界面打磨阶段。

当前八字模块已经具备：

- 基础排盘能力
- UI 基础结构

## 分析重点

重点检查：

### 一、准确性风险

例如：

- 子时问题
- 真太阳时/平太阳时
- 节气边界
- 外部排盘差异
- 术语歧义

---

### 二、UI / UX 问题

例如：

- 信息密度
- 标签重复
- 技术解释过多
- 视觉统一性
- 与主布局基线冲突

---

### 三、架构 / 文档问题

例如：

- 文件结构混乱
- 验证逻辑分散
- 规则缺失
- 历史方案冲突
