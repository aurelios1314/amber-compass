# Task 013：右侧操作区主次层级实现

## 背景

Task 011 与 Task 012 已完成八字模块产品体验评审。

当前结论：

- 八字模块不需要大范围 UI 重做
- 当前核心问题不是布局错误，而是产品层级不清晰
- 右侧操作区中：
  - 生辰输入 / 开始排盘
  - 保存命盘
  - 历史档案
  当前视觉与交互优先级过于接近

目标是：

强化主流程，
降低次级流程干扰，
但不破坏现有布局基线。

---

## 当前阶段

Phase 2：八字模块验证与界面打磨 / 产品体验统一

---

## 任务目标

在不改变整体布局结构的前提下：

调整右侧操作区视觉与交互层级。

目标：

1. 输入生辰 / 开始排盘
   成为明确主流程

2. 保存命盘
   成为排盘后的次级动作

3. 已保存命盘
   成为辅助入口，而不是默认主区域

## 实现原则

### 一、不推翻布局

保持：

- 左果右控
- 固定工作区
- 右侧双列结构

不改为：

- 大面积纵向页面
- 卡片堆叠式玄学页

---

### 二、强化主流程

让用户第一眼明确：

- 在哪里输入
- 如何排盘
- 当前主任务是什么

---

### 三、降低次级流程权重

包括：

- 保存命盘
- 标签
- 备注
- 历史档案

不删除，
但减少默认注意力占用。

---

### 四、保持专业感

避免：

- 传统玄学网站感
- 管理后台感
- 复杂控制台感

保持：

- 克制
- 清晰
- 稳定
- 编辑器 / 工作台气质

## 验收标准

任务完成后：

- 用户第一眼能明确“输入并排盘”是主任务
- 保存与历史不会抢占主流程注意力
- 不破坏整体布局基线
- 不修改业务逻辑
- 八字模块整体更像专业工具，而不是管理后台

---

## 执行要求

1. 先输出实现计划
2. 不直接大改 UI
3. 优先小步调整
4. 修改后自行检查：
   - 布局是否仍稳定
   - 是否出现滚动问题
   - 是否破坏左侧结果区
5. 完成后更新 PROJECT_STATUS 与 TASK_QUEUE
