# Task 017：Amber Compass Empty State System

## 背景

Amber Compass 当前已经完成：

- 基础布局稳定
- UI Token 初步统一
- 视觉方向明确
- Logo 已成熟
- 八字模块主次关系初步稳定

当前产品的问题已经不再是：

“功能是否可用”。

而是：

产品缺少统一的品牌呼吸感。

目前：

空状态仍偏：

- 骨架屏
- 普通工具占位
- 通用后台系统

还没有真正体现：

Amber Compass 的品牌气质。

---

## 当前阶段

Phase 2：视觉语言统一 / 品牌感建立

---

## 任务目标

建立 Amber Compass 的统一空状态系统。

目标：

让：

- Calendar
- Almanac
- Bazi
- Settings
- 初始状态
- 无数据状态

共享同一套：

安静
克制
可信
温润
现代东方

的空状态语言。

---

# 核心方向

Amber Compass 的空状态：

应该像：

“等待用户输入的东方工作台”。

而不是：

- 骨架屏
- 插画页
- 营销页
- 玄学海报
- 传统命理站

---

# 视觉方向

允许：

- 极轻点阵
- 极淡圆环
- 微结构线
- 琥珀定位点
- 极弱几何秩序感

方向参考：

Logo 中的：

- 双圆环
- 河图洛书点阵
- 东方秩序结构

但：

只能抽象转译。

---

# 禁止事项

禁止：

- 八卦背景
- 云纹贴图
- 罗盘 UI 化
- 发光
- 粒子
- 大插画
- 吉祥物
- 玄学符号墙
- 大面积 decorative pattern
- 赛博东方

禁止：

“更高级算命网站”方向。

---

# 重点分析区域

## 一、八字模块空状态

重点确认：

- 当前是否太像骨架屏
- 是否缺少当前模块气质
- 是否缺少可信感
- 是否缺少工作台感

---

## 二、Calendar / Almanac 空状态

重点确认：

- 是否仍偏普通系统
- 是否缺少品牌语言
- 是否与 Logo 割裂

---

## 三、全局空状态语言

重点确认：

- 点阵如何统一
- 圆环如何轻量化
- 留白如何控制
- 提示文案如何克制

---

# 输出要求

输出：

# Amber Compass Empty State Review

包含：

## 当前空状态整体评价

## 当前最大问题

## 哪些模块最割裂

## 哪些地方最缺品牌感

## 哪些元素适合进入空状态系统

## 哪些元素绝对不要进入

## 空状态的正确气质

## 建议优先优化顺序

## 哪些地方不要动

---

# 验收标准

任务完成后：

- Amber Compass 拥有统一空状态方向
- Logo 气质开始进入产品
- 空状态不再像普通后台系统
- 不进入玄学化
- 不破坏现有布局

---

# 执行要求

1. 先输出分析计划
2. 不修改代码
3. 不输出实现代码
4. 不直接设计 UI
5. 先做空状态系统评审
6. 等待确认后继续
